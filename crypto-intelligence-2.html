🔥 Firebase Integration Guide

Follow these steps to connect a real backend — replaces all localStorage with cloud storage.

Step 1 — Create Firebase Project

Go to console.firebase.google.com → New project → name it "cryptoiq" → disable Google Analytics (optional).

Step 2 — Enable Authentication

Build → Authentication → Get started → Sign-in method → Email/Password → Enable. This replaces the demo login.

Step 3 — Create Firestore Database

Build → Firestore Database → Create → Start in test mode. Create 3 collections: airdrops, posts, users.

Step 4 — Get Config Keys

Project settings (⚙️) → Your apps → Web app → Register → copy the firebaseConfig object. Add this to the <head> of your HTML:

import { initializeApp } from "firebase/app";
import { getAuth } from "firebase/auth";
import { getFirestore } from "firebase/firestore";
const app = initializeApp(firebaseConfig);
export const auth = getAuth(app);
export const db = getFirestore(app);
Step 5 — Replace loginUser()

Swap the demo login with:

import { signInWithEmailAndPassword } from "firebase/auth";
async function loginUser() {
  const email = document.getElementById('loginEmail').value;
  const pass  = document.getElementById('loginPass').value;
  try {
    const cred = await signInWithEmailAndPassword(auth, email, pass);
    currentUser = { email: cred.user.email, name: email.split('@')[0] };
    applyUserState();
  } catch(e) { showToast('❌ ' + e.message); }
}
Step 6 — Save Airdrops to Firestore

Replace the airdrops.unshift() line in addAirdrop() with:

import { collection, addDoc, getDocs } from "firebase/firestore";
// Save:
await addDoc(collection(db, "airdrops"), newAirdrop);
// Load all:
const snap = await getDocs(collection(db, "airdrops"));
airdrops = snap.docs.map(d => ({id:d.id, ...d.data()}));

🔒 Security Integration Guide

✅ Firestore Security Rules

Paste this in Firebase → Firestore → Rules tab to restrict access:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Anyone can read airdrops/posts
    match /airdrops/{doc} { allow read: if true; allow write: if request.auth != null && request.auth.token.admin == true; }
    match /posts/{doc}    { allow read: if true; allow write: if request.auth != null && request.auth.token.admin == true; }
    // Users can only access their own data
    match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; }
  }
}
✅ Enable Firebase App Check

Firebase Console → App Check → Register your app with reCAPTCHA v3. This blocks bots and unauthorized API calls from hitting your Firestore database.

✅ Input Sanitization

All admin inputs are trimmed. For production, install DOMPurify: npm i dompurify and wrap content in DOMPurify.sanitize(input) before rendering.

✅ Never Expose Keys in GitHub

When deploying to Vercel/Netlify, store Firebase config as environment variables (VITE_FIREBASE_API_KEY etc.) and access them as import.meta.env.VITE_.... Add .env to your .gitignore.

✅ Comment Spam Protection

For production comments, add Google reCAPTCHA v3 to the comment form or enable Firebase App Check, which covers Firestore writes automatically.
